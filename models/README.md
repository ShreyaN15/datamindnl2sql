# Models Directory

This directory is for ML model artifacts but is **excluded from git** due to large file sizes.

## Model Location

The actual trained T5 NL2SQL model is located at:
```
datamind_t5/final_finetuned/
```

This directory contains:
- `model.safetensors` - Trained model weights (~850 MB)
- `config.json` - Model configuration
- `tokenizer_config.json` - Tokenizer settings
- `spiece.model` - SentencePiece vocabulary
- Other model files

## Usage

The application automatically loads the model from `datamind_t5/final_finetuned/` when making NL2SQL predictions.

No additional setup is required - just ensure the `datamind_t5/final_finetuned/` directory exists with the trained model files.

## Training Your Own Model

To train the model from scratch:

```bash
cd datamind_t5
python train.py
```

This will create a new fine-tuned model in `datamind_t5/workspace/checkpoints/final/`

## Why Models Are Gitignored

Model files are excluded from git because:
- They are very large (850+ MB)
- GitHub has a 100 MB file size limit
- Model binaries don't benefit from version control
- They can be regenerated by training

For production deployments, consider:
- Storing models in cloud storage (S3, GCS, Azure Blob)
- Using model registries (MLflow, DVC, Weights & Biases)
- Downloading models during deployment

## Note

This directory (`models/`) is kept for backwards compatibility but the application uses `datamind_t5/final_finetuned/` as the primary model location.
